# NEMU-RV32: Execute ADDI

## [韌體筆記]
### 保護 x0 (Zero Register)
在 RISC-V 架構中，`x0` 暫存器被硬體寫死（Hardwired）為 `0`。
- **目的：** 簡化指令集設計。許多操作可以透過 `x0` 簡化，例如 `MV rd, rs` 可以實作為 `ADDI rd, rs, 0`，而 `NOP` 可以實作為 `ADDI x0, x0, 0`。
- **實作：** 在模擬器中，我們必須在寫回暫存器之前檢查 `rd != 0`。即使指令嘗試寫入 `x0`，其值也必須保持為 `0`。

## [CSAPP 筆記]
### 符號擴展 (Sign Extension) 與 2 的補數
- **符號擴展：** 當我們將一個較短的有號數（如 12-bit `imm`）轉換為較長的格式（如 32-bit `int32_t`）時，必須複製原始數值的最高有效位元（Sign Bit）到所有新增的高位元。
- **原理：** 這保證了在 2 的補數（Two's Complement）表示法下，數值的正負號與大小保持不變。
- **C 語言技巧：** 透過 `(res << (32 - 12)) >> (32 - 12)`，利用有號整數的算術右移（Arithmetic Right Shift）特性，可以高效地完成符號擴展。
