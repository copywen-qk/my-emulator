# NEMU-RV32: MMIO, LUI & SB

## [韌體筆記]
### Memory-Mapped I/O (MMIO)
- **概念：** MMIO 將外設（如串口、計時器、顯示卡）的寄存器映射到與普通記憶體相同的位址空間。CPU 不需要特殊的指令，只需使用標準的 `Load/Store` 指令即可與硬體通訊。
- **volatile 的必要性：** 在 C 語言開發真實韌體時，存取 MMIO 位址必須使用 `volatile` 關鍵字。這告訴編譯器該位址的值可能在程式不知情的情況下改變（由硬體改變），或者寫入該位址有副作用（如觸發硬體動作），從而防止編譯器將其優化掉。

## [CSAPP 筆記]
### 系統匯流排 (System Bus) 與 I/O 路由
- **匯流排的角色：** 當 CPU 執行存取記憶體的指令（如 `SB`）時，位址與資料會被送往系統匯流排。
- **I/O 控制器：** 匯流排上有位址解碼邏輯。如果位址落在實體 DRAM 的範圍，請求會被送往記憶體控制器；如果位址落在特定的 I/O 範圍（如我們模擬的 `0xa00003f8`），請求則會被路由到對應的 I/O 裝置控制器（如 UART）。
- **NEMU 實作：** 在我們的模擬器中，`paddr_write` 函數充當了「匯流排控制器」的角色，負責將記憶體存取請求分流到 `pmem` 陣列或 `putchar` 函數。
